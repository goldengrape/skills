# Literate Architecture Record (LAR) Patterns

The LAR is the final deliverable. It should feel like a technical essay.

## LAR Template

```markdown
# [Project Name] Literate Architecture Record

## 1. Intent & Context
[Describe the problem this project solves and its design philosophy.]

## 2. Core Domain Map
[Describe the primary entities and their relationships. Use a Mermaid diagram if helpful.]

## 3. The Guided Tour
[Step-by-step walkthrough of the most important logic paths.]

### Example: [Module Name]
> Quote from code: "..."
[Prose explanation of the logic.]
```python
# Key logic snippet
[Code]
```

## 4. Architectural Decisions (ADR)
[List key decisions made and why, referencing the LP-skill philosophy.]

## 5. Verification & Proof of Work
[Summarize how the system was tested or verified.]
```
